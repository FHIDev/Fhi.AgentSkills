# Delta: v0.34.x

## Scope

Gjelder v0.34.x (alle patcher).
Skrevet mot: v0.35.0. Innhold er kumulativt — gyldige avvik vs. enhver nyere latest.

## Missing vs latest

### fhi-text-input

- Mangler slottene `start` og `end` for ikon-plassering. Disse ble innført i v0.35.0
  og lar deg legge et FHI-ikon på venstre eller høyre side av inputfeltet.

```html
<!-- Krever v0.35.0+: -->
<fhi-text-input label="Brukernavn">
  <fhi-icon-user slot="start"></fhi-icon-user>
</fhi-text-input>
```

I v0.34.x må visuell ikon-markering håndteres utenfor komponenten (f.eks. som
egen layout rundt `fhi-text-input`).

### Data Table

- Hele komponentgruppen `fhi-data-table`, `fhi-data-table-row` og
  `fhi-data-table-cell` mangler (innført i v0.37.0).

### fhi-tag

- Mangler `variant="bordered"` (innført i v0.40.0).

### Ikoner

- Mangler `fhi-icon-file-text` (innført i v0.38.0).

## Different vs latest

### Typografikomponenter — default color

- Default `color` er `var(--fhi-color-neutral-text-default)`, **ikke** `currentcolor`.
  Latest endret default til `currentcolor` (arver tekstfarge fra forelder) i v0.39.0.

### fhi-text-input

- `<input>`-elementet i v0.34.x har **ikke** `flex: 1 1 auto` på input-elementet.
  v0.35.0 la til denne regelen som del av "Fix behaviour of input width"-fiksen.
  Konsekvens: i flex-layouts der `fhi-text-input` må strekke seg eller dele plass
  med søsken, kan bredde-oppførselen avvike fra v0.35.0. Test layout etter
  oppgradering hvis appen har egne bredde-styles eller flex-baserte input-rader.

## Legacy-only

Ingen.

## Import/entrypoints

Runtime-importstier er uendret. Publiserte `exports`-entrier og `.d.ts`-filer per
entrypoint er på plass (innført i v0.34.0). `theme/default.css` er byte-identisk
med v0.35.0.

## Theme/tokens

Ingen endringer. `theme/default.css` er byte-identisk mellom v0.34.0 og v0.35.0.

## Rammeverk-notater (Blazor)

Ingen avvik fra latest.

## Patch notes med API-impact

Ingen kjente patches med API-endringer (kun v0.34.0 publisert i 0.34-serien).

## Migrering (0.34 → 0.35)

- Bakoverkompatibel oppgradering. Ingen kodeendringer er nødvendige.
- Hvis du vil bruke ikon i `fhi-text-input`: oppgrader til v0.35.0 og bruk
  slottene `start`/`end` med en `fhi-icon-*`-komponent.
- Test layout der `fhi-text-input` brukes i flex-baserte rader, fordi
  bredde-oppførselen er endret av "Fix behaviour of input width"-fiksen.

## Manifest-merknad

De nye `start`/`end`-slotene på `fhi-text-input` reflekteres **ikke** i
`custom-elements.json` eller `web-types.json` i v0.35.0 — manifestene viser
fortsatt tom slot-liste for komponenten. Slot-støtten er verifisert via
runtime-koden (`fhi-text-input.component.ts`), Storybook-docs (`.docs.mdx`) og
stories (`.stories.ts`). Kontrollsjekker basert utelukkende på manifests vil
derfor ikke fange opp denne featuren.
